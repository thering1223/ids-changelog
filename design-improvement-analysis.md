# INPOCK 딜 시안 개선점 분석

## 개요
"셀렉티드 캠페인" 대시보드 페이지의 UX/UI 개선점을 분석한 문서입니다.

---

## 🔴 긴급 개선 사항 (Critical Issues)

### 1. 테이블 정보 과부하 (Information Overload)
**문제점:**
- 테이블에 **13개 이상의 컬럼**이 한 화면에 표시됨
- 가로 스크롤이 필요하거나 화면이 좁은 경우 정보 접근성 저하
- 사용자가 핵심 정보를 빠르게 파악하기 어려움

**개선 방안:**
- **컬럼 가시성 토글**: 사용자가 필요한 컬럼만 선택하여 표시
- **기본 컬럼 그룹화**: 필수 컬럼(상태, 캠페인 정보, 카테고리)만 기본 표시
- **상세 보기 모드**: 행 클릭 시 모달/드로어에서 전체 정보 확인
- **컬럼 우선순위 재정의**: 자주 사용하는 컬럼을 앞쪽으로 배치

### 2. 필터/검색 기능 부재
**문제점:**
- "별 고정선" 텍스트만 있고 실제 필터 UI가 없음
- 300만 인플루언서 데이터에서 원하는 캠페인을 찾기 어려움
- 검색 기능이 보이지 않음

**개선 방안:**
- **통합 검색바 추가**: 캠페인명, 카테고리, 크리에이터명 등으로 검색
- **필터 패널 구현**:
  - 상태별 필터 (모집 중, 종료, 정산 완료 등)
  - 카테고리 필터
  - 크리에이터 티어 필터
  - 날짜 범위 필터
- **저장된 필터**: 자주 사용하는 필터 조합 저장 기능
- **정렬 옵션**: 날짜, 인기순, 상태별 정렬

### 3. 상태 태그 일관성 부족
**문제점:**
- 상태 태그가 **5가지 이상의 색상**으로 표시됨 (회색, 초록, 주황, 파랑, 빨강)
- 색상 의미가 명확하지 않음
- 같은 색상이 다른 의미로 사용될 수 있음

**개선 방안:**
- **상태 색상 체계 정립**:
  - 🔵 파란색: 진행 중/활성 상태
  - 🟢 초록색: 완료/성공 상태
  - 🟡 노란색: 대기/주의 상태
  - 🔴 빨간색: 오류/중지 상태
  - ⚪ 회색: 비활성/종료 상태
- **상태 아이콘 추가**: 색상만으로는 구분이 어려울 수 있으므로 아이콘 병행
- **상태 설명 툴팁**: 각 상태에 마우스 오버 시 설명 표시

## 🟡 중요 개선 사항 (High Priority)

### 4. 테이블 행 정보 가독성
**문제점:**
- "Untitled" 캠페인이 표시됨 (데이터 품질 문제)
- 캠페인 정보 컬럼에 이미지와 텍스트가 혼재되어 가독성 저하
- 콘텐츠 제작 방식 등 일부 컬럼의 텍스트가 길어 잘림

**개선 방안:**
- **데이터 검증**: 필수 필드 미입력 시 저장 방지 또는 경고 표시
- **텍스트 말줄임 처리**: 긴 텍스트는 말줄임표(...) 처리 후 툴팁으로 전체 내용 표시
- **이미지 크기 최적화**: 썸네일 크기 통일 및 로딩 상태 표시
- **행 높이 조정**: 정보 밀도와 가독성의 균형

### 5. 액션 버튼 위치 및 명확성
**문제점:**
- "캠페인 등록" 버튼이 우측 상단에만 있음
- 각 행의 "관리" 버튼이 회색으로 덜 눈에 띔
- "더보기(...)" 버튼의 기능이 불명확

**개선 방안:**
- **플로팅 액션 버튼(FAB)**: 우측 하단에 "캠페인 등록" 버튼 추가 고려
- **행 액션 메뉴 개선**: "더보기" 클릭 시 드롭다운 메뉴로 명확한 액션 표시
  - 수정
  - 복제
  - 삭제
  - 상태 변경
- **관리 버튼 스타일 개선**: 더 눈에 띄는 스타일 또는 아이콘 추가

### 6. 숫자 데이터 시각화 부족
**문제점:**
- 클릭/신청 수, 선정 인원 등이 단순 숫자로만 표시됨
- 트렌드나 비교가 어려움
- 중요 지표가 강조되지 않음

**개선 방안:**
- **미니 차트 추가**: 일별/주별 트렌드를 작은 스파크라인으로 표시
- **색상 코딩**: 목표 대비 달성률에 따라 색상 변화
- **비교 표시**: 전일 대비 증감률 표시 (예: +15% ↑)
- **핵심 지표 카드**: 테이블 상단에 요약 카드 추가

### 7. 페이지네이션 개선
**문제점:**
- 페이지 번호만 표시되어 페이지 크기나 전체 개수 파악 어려움
- 페이지 이동이 번거로울 수 있음

**개선 방안:**
- **페이지 크기 선택**: 10개/20개/50개/100개 선택 옵션
- **전체 개수 표시**: "총 120개 중 1-10개 표시"
- **빠른 이동**: 첫 페이지/마지막 페이지 바로가기 버튼
- **무한 스크롤 옵션**: 선택적으로 무한 스크롤 모드 제공

## 🟢 개선 제안 사항 (Enhancement Suggestions)

### 8. 반응형 디자인
**개선 방안:**
- 모바일/태블릿에서 테이블을 카드 형태로 변환
- 핵심 정보만 표시하고 상세 정보는 확장 가능한 형태로 구성

### 9. 빈 상태(Empty State) 처리
**개선 방안:**
- 필터 결과가 없을 때 안내 메시지 및 액션 제안
- 첫 캠페인 등록 시 가이드 제공

### 10. 키보드 단축키 지원
**개선 방안:**
- `/` 키로 검색바 포커스
- `Ctrl/Cmd + K`로 빠른 액션 메뉴
- 화살표 키로 행 이동

### 11. 대량 작업 기능
**개선 방안:**
- 체크박스로 여러 캠페인 선택
- 일괄 상태 변경, 삭제, 내보내기 기능

### 12. 내보내기 기능
**개선 방안:**
- 테이블 데이터를 Excel/CSV로 내보내기
- 현재 필터 적용된 데이터만 내보내기 옵션

### 13. 알림 및 뱃지 개선
**문제점:**
- 뱃지가 작고 숫자만 표시되어 어떤 알림인지 불명확

**개선 방안:**
- 뱃지에 툴팁 추가 (예: "승인 대기 1건")
- 알림 센터 연결: 뱃지 클릭 시 관련 알림으로 이동
- 색상으로 우선순위 표시

### 14. 접근성(Accessibility) 개선
**개선 방안:**
- ARIA 레이블 추가
- 키보드 네비게이션 지원
- 색상 대비율 개선 (WCAG 2.1 AA 기준 준수)
- 스크린 리더 지원

## 🎨 디자인 토큰 활용 제안

현재 사용 중인 디자인 토큰:
- Primary: `#ff5b1a` (orange/800)
- Gray Scale: gray/100 ~ gray/800
- Typography: Pretendard, Spoqa Han Sans Neo, Apple SD Gothic Neo

**제안:**
- 상태별 색상 토큰 추가 정의
- 간격(Spacing) 토큰 체계화
- 그림자(Shadow) 토큰 추가

## 📝 결론

현재 시안은 기능적으로는 완성도가 높지만, **사용성과 정보 구조 측면에서 개선이 필요**합니다. 특히:

1. **정보 과부하 해결**이 가장 시급합니다
2. **필터/검색 기능**은 필수입니다
3. **상태 표시의 일관성**이 필요합니다

이러한 개선을 통해 사용자가 300만 인플루언서 데이터를 효율적으로 관리하고 원하는 캠페인을 빠르게 찾을 수 있을 것입니다.
